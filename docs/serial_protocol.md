# 串口协议说明

本项目中，视觉/上位机模块通过 **USART1** 向小车下位机发送单字节命令。

## 数据包格式

```text
0xAA + CMD + 0xFF
```

- `0xAA`：包头
- `CMD`：命令字节
- `0xFF`：包尾

## 命令定义

| CMD | 含义 | 下位机行为 |
| --- | --- | --- |
| `0x00` | 检测到菱形减速区 | 进入减速流程 |
| `0x01` | 检测到人行横道 | 进入停车/等待流程 |
| `0x02` | 检测到左转标志 | 记录左转任务 |
| `0x03` | 检测到右转标志 | 记录右转任务 |
| `0x04` | 检测到红色终点线 | 触发停车逻辑 |
| `0x05` | 开始运行 | 状态机进入启动状态 |

## 说明

- 当前协议只解析 **1 字节有效载荷**，也就是一个 `CMD`。
- 协议接收由 `Hardware/Serial.c` 中的 `USART1_IRQHandler()` 状态机实现。
- 如果后续要扩展更多参数，可以把当前的单字节命令改为“命令字节 + 数据区 + 校验位”的形式。
