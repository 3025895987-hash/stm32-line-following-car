# STM32 循迹小车下位机工程

这是一个基于 **STM32F103C8** 的循迹小车下位机工程，使用 **Keil MDK** 与 **STM32 Standard Peripheral Library** 开发。

项目包含电机驱动、舵机转向、超声波测距、JY61P 姿态角读取，以及与视觉/上位机之间的串口通信逻辑。整体控制采用“**状态机 + 串口事件触发 + 偏航角 PID 修正**”的思路实现。

## 这是一个什么项目

这个仓库主要负责小车底盘侧控制，也就是常说的 **下位机代码**。

它做的事情包括：

- 初始化电机、舵机、超声波、OLED、串口、姿态模块
- 接收视觉模块通过串口发来的识别结果
- 根据识别结果切换不同的运动状态
- 使用 JY61P 提供的偏航角，结合 PID 对方向进行修正
- 在不同赛道元素下执行直行、转弯、减速、停车等动作

## 功能概览

- **直行控制**：通过 PID 将目标偏航角与当前偏航角对齐
- **转向动作**：通过设置舵机角度完成左转/右转过程
- **超声波测距**：在部分阶段利用 HC-SR04 判断距离条件
- **串口协议解析**：接收视觉模块发送的命令包
- **终点停车**：识别到红线命令后进入停车流程

## 核心控制思路

主流程位于 `User/main.c`。

程序上电后会先完成：

- 底盘与舵机初始化
- OLED 初始化
- USART1 初始化（视觉/上位机）
- USART3 初始化（JY61P）
- 偏航角 PID 初始化

随后主循环中主要做两件事：

1. 持续读取 JY61P 的相对偏航角并显示。
2. 根据 `mode` 状态机执行不同行为：
   - 直行
   - 左转/右转
   - 菱形减速
   - 人行横道停车等待
   - 识别 L / R 标志后的动作
   - 红线停车

## 串口协议

视觉模块通过 **USART1** 发命令给小车，下位机接收的数据包格式为：

```text
0xAA + CMD + 0xFF
```

命令定义如下：

| 命令 | 含义 |
| --- | --- |
| `0x00` | 检测到菱形减速区 |
| `0x01` | 检测到人行横道 |
| `0x02` | 检测到左转标志 |
| `0x03` | 检测到右转标志 |
| `0x04` | 检测到红色终点线 |
| `0x05` | 小车启动 |

更详细的说明见：[`docs/serial_protocol.md`](docs/serial_protocol.md)

## 开发环境

- MCU：STM32F103C8
- IDE：Keil MDK
- 外设库：STM32 Standard Peripheral Library
- 姿态模块：JY61P
- 测距模块：HC-SR04

## 如何打开工程

1. 使用 Keil 打开 `Project.uvprojx`
2. 选择对应 Target
3. 检查芯片型号与下载器配置
4. 编译并下载到开发板

> 仓库中没有保留 `Objects/`、`Listings/`、`*.uvoptx`、`*.uvguix.*` 这类构建产物或本机配置文件；这些文件可以由 Keil 在本地重新生成。
